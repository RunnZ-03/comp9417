All datasets were split 60/20/20 into train/validation/test sets with a fixed random seed of 42.
Numerical features were standardised with z-score normalisation, and categorical features were
one-hot encoded. Missing values were imputed using column-wise median (numerical) or the constant
\texttt{"Missing"} (categorical).

only once on the final model.

\subsubsection{XGBoost and LightGBM Configuration}
\label{sec:gbdt-config}

% ═══════════════════════════════════════════════════════════════════════════
%  YOUR SECTION — Written by C (XGBoost / LightGBM person)
% ═══════════════════════════════════════════════════════════════════════════

We train two Gradient Boosted Decision Tree (GBDT) baselines: XGBoost~\citep{chen2016xgboost}
and LightGBM~\citep{ke2017lightgbm}. Both models are trained using the scikit-learn-compatible
API and share a common hyperparameter configuration designed to reflect strong default settings
reported in the literature~\citep{holzmuller2024better}.

\paragraph{Shared hyperparameters.}
For both models, we set the maximum number of boosting rounds to 1{,}000, the learning rate to
$\eta = 0.05$, the row subsampling ratio to $0.8$, and the column subsampling ratio to $0.8$.
Early stopping with a patience of 20 rounds is applied using validation-set RMSE (regression) or
log-loss (classification), preventing overfitting without manual tuning of the number of trees.

\paragraph{XGBoost-specific settings.}
The maximum tree depth is set to 6, which controls model complexity and is consistent with the
default configuration used in large-scale benchmarks~\citep{grinsztajn2022tree}.
XGBoost is trained with the \texttt{hist} tree method and \texttt{verbosity=0}.

\paragraph{LightGBM-specific settings.}
Rather than constraining tree depth, LightGBM is configured with \texttt{num\_leaves=31}, which
directly bounds the number of leaves per tree and better reflects LightGBM's leaf-wise growth
strategy~\citep{ke2017lightgbm}. Verbose output is suppressed via \texttt{verbose=-1}.

\paragraph{Classification labels.}
For datasets with string-valued class labels (e.g.\ \texttt{"Yes"}/\texttt{"No"} in HR Attrition),
labels are integer-encoded via scikit-learn's \texttt{LabelEncoder} prior to training, as both
XGBoost and LightGBM require numeric targets.
