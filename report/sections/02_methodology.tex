\section{Methodology}

\subsection{xRFM: Algorithm and Theoretical Background}

xRFM is a tabular feature-learning model based on the Average Gradient Outer Product (AGOP)~\citep{beaglehole2025xrfm,radhakrishnan2024mechanism}. For a trained prediction function $f:\mathbb{R}^{d}\rightarrow\mathbb{R}$, the AGOP is defined as
\begin{equation}
    G_f=\mathbb{E}_{x}\!\left[\nabla f(x)\nabla f(x)^{\top}\right] \in \mathbb{R}^{d\times d}.
    \label{eq:agop}
\end{equation}
The diagonal entry $[G_f]_{ii}=\mathbb{E}_x[(\partial f/\partial x_i)^2]$ is the expected squared partial derivative with respect to feature $i$: a large value indicates rapid change in the model output along coordinate $i$ (important feature); a small value indicates near-insensitivity. The full $G_f$ supports two interpretability strategies: its \emph{diagonal} gives coordinate-wise importance scores for each feature, while its \emph{top eigenvectors} identify multivariate directions in input space along which the model varies most, potentially capturing signal invisible to any single coordinate. We use diagonal AGOP in our experiments as it is cheaper to store and sufficient for feature-level rankings.

Compared with alternative methods, AGOP is supervised and model-specific: it reflects the gradient geometry of the fitted function, unlike PCA which only uses input covariance~\citep{pearson1901pca}. Mutual information measures marginal feature–target dependence but not the model's multivariate sensitivity~\citep{kraskov2004estimating}. Permutation importance is model-agnostic but measures post-hoc performance degradation rather than learned geometry~\citep{fisher2019all}. AGOP is extracted as a by-product of training and directly summarises which directions the learned model relies on.

\paragraph{Training pipeline.}
xRFM builds a binary tree over the training data. At each node, a base model $f$ is fitted to the local data partition and its AGOP $G_f$ is computed. The leading eigenvector $v_1$ of $G_f$ defines the split direction; the threshold $t^*$ minimises weighted target variance along the projection $v_1^\top x$. Each leaf fits a local random Fourier feature kernel machine with the feature metric reweighted by $G_f$. For inference on a new point $x^*$, the tree is traversed via $v_k^\top x^* \leq t_k$ at each internal node, and the leaf model produces the final prediction. The dominant training cost is AGOP computation: $O(nd)$ per node for a linear approximation, giving total complexity $O(nd\cdot 2^D)$ for tree depth $D$~\citep{beaglehole2025xrfm}. This addresses the fundamental limitation of standard kernel machines on tabular data — that a global Euclidean metric treats all features equally — by adaptively reweighting feature space at each tree node.

\subsubsection{Preprocessing and Splitting}

All experiments use seed 42 and a 60/20/20 split. Numerical features are z-score normalised and categorical features are one-hot encoded; missing values are median/constant imputed. The same encoded matrix is used across all four models.

\subsubsection{Evaluation Metrics}

Regression tasks are evaluated with RMSE; binary classification tasks with accuracy and AUC-ROC. Training and inference time are also recorded.

\subsubsection{xRFM Configuration}
\label{sec:xrfm-config}

The xRFM experiments use the Python \texttt{xrfm} package through the shared project training pipeline. The model is initialised as \texttt{xRFM(use\_diag=True, device=DEVICE)}, where \texttt{DEVICE} is CUDA if available and CPU otherwise. On GPU, the training script sets the number of fit iterations to $3$ and \texttt{M\_batch\_size=128} to control memory cost. Feature importance is extracted by reading the learned AGOP matrix or diagonal values from the trained xRFM model and ranking features by their AGOP-derived scores.

\subsubsection{XGBoost and LightGBM Configuration}
\label{sec:gbdt-config}

XGBoost and LightGBM are trained through the scikit-learn API using strong default settings from recent tabular benchmarking work~\citep{chen2016xgboost,ke2017lightgbm,holzmuller2024better}. Both use 1{,}000 maximum boosting rounds, learning rate $\eta=0.05$, row and column subsampling of 0.8, and early stopping with patience 20 on the validation set. XGBoost uses maximum tree depth 6, while LightGBM uses \texttt{num\_leaves=31}.

\subsubsection{MLP Configuration}
\label{sec:mlp-config}

The MLP baseline uses a two-hidden-layer network with hidden sizes $(128,64)$ and ReLU activation. It is trained with Adam, learning rate 0.001, batch size 256, maximum 500 iterations, and early stopping with patience 20. Input features are standardised before training. For classification tasks, predicted probabilities are used for AUC-ROC calculation.
