\section{Methodology}

\subsection{xRFM: Algorithm and Theoretical Background}

xRFM learns task-relevant feature directions by iteratively estimating the Average Gradient Outer Product (AGOP)~\citep{beaglehole2025xrfm,radhakrishnan2024mechanism}. For a trained model $f:\mathbb{R}^{d}\rightarrow\mathbb{R}$, the AGOP is
\begin{equation}
    G_f = \mathbb{E}_{x}\!\left[\nabla f(x)\nabla f(x)^{\top}\right] \in \mathbb{R}^{d\times d},
    \label{eq:agop}
\end{equation}
where large diagonal entries of $G_f$ indicate features to which $f$ is most sensitive. Unlike PCA (unsupervised input-variance directions~\citep{pearson1901pca}), mutual information (univariate target dependence~\citep{kraskov2004estimating}), or permutation importance (post-hoc performance drop~\citep{fisher2019all}), AGOP captures supervised, multivariate sensitivity arising directly from the model's gradients. The diagonal entries provide coordinate-wise feature importance; the leading eigenvectors reveal task-relevant linear combinations of inputs.

The xRFM training pipeline iterates: (1) fit a kernel/random-feature model on the current feature space; (2) estimate $G_f$ from training gradients; (3) reweight the feature space by $G_f$ to focus on predictive directions; repeat for a fixed number of iterations. Inference routes a new point $x^*$ through the learned adaptive tree to the appropriate leaf model. We use diagonal AGOP (\texttt{use\_diag=True}) throughout to reduce memory cost from $O(d^2)$ to $O(d)$ while preserving feature-level interpretability.

\subsubsection{Data Preprocessing and Splitting Strategy}

To maintain fairness and consistency across all experiments, all random seeds were fixed at 42 for reproducibility. For each dataset, we applied a strict 60/20/20 split for the training, validation, and test sets. Missing values were imputed using the column-wise median for numerical features and the constant \texttt{Missing} for categorical features. Numerical features were standardised using z-score normalisation, and categorical features were transformed using one-hot encoding. The processed features were concatenated into a single numerical array before being passed to all models, ensuring compatibility with xRFM's AGOP computations and with the baseline models.

\subsubsection{Evaluation Metrics}

Regression tasks are evaluated by RMSE (lower is better); classification tasks by accuracy and AUC-ROC, where AUC is preferred under class imbalance. We also record training time and per-sample inference time for all models. All hyperparameters are selected on the validation set; test metrics are computed once on the held-out test set.

\subsubsection{xRFM Configuration}
\label{sec:xrfm-config}

The xRFM model was implemented using the Python \texttt{xrfm} package and trained through the shared project pipeline. The training script initialises the model as \texttt{xRFM(use\_diag=True, device=DEVICE)}, where \texttt{DEVICE} is set to CUDA when available and CPU otherwise. The diagonal setting was selected because it substantially reduces the memory cost of AGOP estimation while producing feature-level importance values. The project code also fixes NumPy and PyTorch random seeds at 42, which improves reproducibility across repeated runs~\citep{pedregosa2011scikit,paszke2019pytorch}.

When running on GPU, the code reduces the number of xRFM fit iterations to $3$ and sets \texttt{M\_batch\_size=128}. This setting was used as a practical compromise between computational cost and model quality. All xRFM models were fitted on the training split, monitored using the validation split, and evaluated once on the held-out test split. For classification tasks, the code used predicted class labels for accuracy and attempted to use \texttt{predict\_proba} scores for AUC-ROC. For regression tasks, predictions were evaluated directly using RMSE. Feature importance was extracted from the learned model by reading the AGOP matrix or diagonal values from the internal xRFM tree model and ranking features by their AGOP-derived scores.

\subsubsection{XGBoost and LightGBM Configuration}
\label{sec:gbdt-config}

Both models are trained via the scikit-learn API following strong-default settings~\citep{holzmuller2024better}: 1{,}000 boosting rounds, $\eta=0.05$, row/column subsample~0.8, and early stopping patience~20 on validation loss. XGBoost uses max depth~6; LightGBM uses \texttt{num\_leaves=31} to match its leaf-wise growth strategy~\citep{ke2017lightgbm}.

\subsubsection{MLP Configuration}
\label{sec:mlp-config}

The MLP baseline uses a two-hidden-layer network of sizes $(128, 64)$ with ReLU activations, trained with Adam ($\eta=0.001$, batch size 256, max 500 epochs) and early stopping with patience~20 on validation loss. All features are z-score standardised; classification outputs use softmax probabilities for AUC computation.
